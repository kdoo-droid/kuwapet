# KuwaPet Website Docs

This folder is a full planning pack for building `kuwapet.com`.

The goal is to give Claude enough strategic, creative, technical, and compliance context to design and build a strong ecommerce website for a dog meal-topper brand centered on mulberry powder.

## Project Snapshot

- Brand: `KuwaPet`
- Domain: `kuwapet.com`
- Product type: canine meal topper / wellness-forward dog product
- Core ingredient concept: mulberry powder for dogs
- Launch model: single-product direct-to-consumer ecommerce site
- Market assumption: United States
- Current stage: pre-build, strategy and documentation only

## Important Assumptions

These docs make a few reasonable assumptions so Claude can move forward without waiting on missing details:

- KuwaPet launches with one hero product first.
- The public-facing website should prefer the language `dog meal topper` or `daily topper` over `supplement` unless regulatory review confirms stronger category language.
- The website should feel premium, warm, modern, and trustworthy, not clinical or gimmicky.
- The site should avoid unsubstantiated health claims.
- Missing factual product details such as price, bag size, certifications, feeding directions, and sourcing specifics should be treated as placeholders until confirmed.

## Website Objectives

The KuwaPet website should:

- Make the brand feel credible within a few seconds.
- Explain what the product is without relying on hype.
- Make mulberry feel intentional and intriguing.
- Convert first-time visitors into buyers.
- Work beautifully on mobile.
- Leave room for future expansion into more products, bundles, education, and subscriptions.

## Non-Negotiables

- Keep copy emotionally strong but claim-safe.
- Do not fabricate certifications, studies, reviews, or veterinarian endorsements.
- Do not imply that the product treats, cures, or prevents disease.
- Use placeholders clearly where real business data is still missing.
- Design the site for both desktop and mobile from the start.
- Build the brand around trust, simplicity, and ritual, not miracle promises.

## Recommended Build Order

If Claude is going from docs to implementation, the recommended order is:

1. Set up the global design system and layout shell.
2. Build the homepage and primary product section.
3. Build the product page and FAQ experience.
4. Add About, Why Mulberry, Contact, and policy-ready utility pages.
5. Add SEO metadata, schema, analytics hooks, and accessibility polish.
6. Replace placeholders with real commerce data when available.

## Missing Inputs To Confirm Before Launch

- Final product name
- Final package size and unit format
- Final price and shipping thresholds
- Ingredient statement and guaranteed analysis, if applicable
- Feeding directions
- Storage instructions
- Manufacturer / distributor information
- Ingredient sourcing details
- Return policy
- Any real reviews, UGC, or veterinarian relationships
- Any substantiated claims legal counsel approves

## Working Principle

If there is tension between "high converting marketing copy" and "truthful, durable brand trust," choose trust.
